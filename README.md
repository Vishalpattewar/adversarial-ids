# Adversarial Input Attack on ML-based IDS
### CDAC ITISS Project

---

## Overview

This project implements and evaluates adversarial attacks on a Neural Network-based
Intrusion Detection System (IDS) trained on the **NSL-KDD** dataset.

Two models are trained and compared:
- **Standard IDS** — trained on clean data only (no defence)
- **Adversarial IDS** — hardened using FGSM adversarial training

Both models are then attacked using:
- **FGSM** — Fast Gradient Sign Method (single-step attack)
- **PGD** — Projected Gradient Descent (iterative attack, stronger)

A **live real-time IDS** with a Rich terminal dashboard monitors
actual network traffic and raises alerts when attacks are detected.

## Setup Instructions

### Step 1 — Clone Repository
```bash
git clone https://github.com/YOUR_USERNAME/adversarial-ids.git
cd adversarial-ids
```

### Step 2 — Create Virtual Environment
```bash
python3 -m venv venv
source venv/bin/activate
```

### Step 3 — Install Dependencies
```bash
pip install -r requirements.txt
pip install torch --index-url https://download.pytorch.org/whl/cpu
```

### Step 4 — Download NSL-KDD Dataset
```bash
mkdir -p data/kdd

wget "https://raw.githubusercontent.com/jmnwong/NSL-KDD-Dataset/master/KDDTrain%2B.txt" \
     -O data/kdd/KDDTrain+.txt

wget "https://raw.githubusercontent.com/jmnwong/NSL-KDD-Dataset/master/KDDTest%2B.txt" \
     -O data/kdd/KDDTest+.txt
```

### Step 5 — Prepare Dataset (Run Once)
```bash
python3 prepare_kdd.py
```

### Step 6 — Run Full Experiment
```bash
cd src
python3 run_experiment.py
```

### Step 7 — Generate Charts
```bash
python3 visualise_results.py
```

### Step 8 — View Charts (WSL)
```bash
explorer.exe ../results/plots/
```

## Running Live IDS

#### Terminal 1 — Start Live Monitor
```bash
cd src
sudo /home/$USER/A-IDS/adversarial_ids/venv/bin/python3 live_ids.py
```

#### Terminal 2 — Generate Test Traffic
```bash
cd ~/A-IDS/adversarial_ids
sudo /home/$USER/A-IDS/adversarial_ids/venv/bin/python3 test_traffic.py
```

Choose traffic scenario:

```
1 → Normal traffic only
2 → Attack traffic only   (triggers alerts)
3 → Mixed traffic         (most realistic)
4 → Continuous loop       (best for presentation)
```

## Results — NSL-KDD Dataset

### Model Accuracy

| Model                        | Accuracy |
|------------------------------|----------|
| Standard IDS (No Defence)    | 98.17%   |
| Adversarially Hardened IDS   | 97.60%   |

### Evasion Rates

| ε    | Std FGSM | Std PGD | Adv FGSM | Adv PGD | Defence Gain |
|------|----------|---------|----------|---------|--------------|
| 0.05 | 1.3%     | 1.4%    | 0.8%     | 0.9%    | +0.5%        |
| 0.50 | 16.8%    | 19.6%   | 5.1%     | 6.5%    | +11.7%       |
| 0.75 | 56.0%    | 60.2%   | 7.5%     | 9.0%    | +48.5%       |
| 1.00 | 84.7%    | 85.9%   | 20.4%    | 15.0%   | +64.3%       |
| 1.50 | 85.0%    | 88.6%   | 61.0%    | 49.2%   | +24.0%       |
| 2.00 | 85.0%    | 88.6%   | 36.8%    | 74.6%   | +48.2%       |

### Key Findings

- Adversarial training reduces evasion rate by **64.3%** at ε=1.0
- Only **0.57%** accuracy trade-off for significant robustness gain
- PGD attack is stronger than FGSM at all epsilon values
- Live IDS successfully detects SYN floods, port scans and UDP floods

## Architecture

```
Input (38 features)
      ↓
 BatchNorm
      ↓
 Dense(256) → ReLU → Dropout(0.3)
      ↓
 Dense(128) → ReLU → Dropout(0.3)
      ↓
 Dense(64)  → ReLU
      ↓
 Dense(1)   → Sigmoid
      ↓
Output: Normal (0) / Attack (1)
```
